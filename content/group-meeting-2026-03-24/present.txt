研究背景与挑战
逆合成任务是给定目标产物预测其前体反应物。现有方法虽然已经很强，但主要依赖带标签反应数据，而这类数据存在规模有限、覆盖不足和标注成本高的问题。作者因此提出一个核心问题：能否从海量无标签分子中学习可迁移的化学知识，用于提升逆合成预测。

核心思路与论文贡献
KnowRetro 的核心思想是把无标签分子变成结构化知识来源。具体来说，作者构建了分子、子结构和官能团三层知识图谱，再通过化学预训练和知识适配器把这些知识注入逆合成模型。整篇论文最重要的贡献，是把“反应监督学习”推进到了“知识引导学习”。

KnowRetro 总体框架
整个框架包括三部分。第一部分是从无标签分子构建层次化 chemical KG。第二部分是分子到子结构序列的预训练。第三部分是在下游逆合成任务中，把任务相关知识注入编码器和解码器。这个设计形成了一条完整的知识流，而不是单独加一个辅助模块。

层次化化学知识图谱
在知识构建阶段，作者先用 BRICS 对分子进行拆分，得到合成相关的子结构，再用 SMARTS 识别官能团，最后形成分子、子结构、官能团三层结构。这样做的价值在于，子结构更接近断键模式，官能团更接近反应活性，两者互补。

反应感知预训练
预训练任务是让模型根据分子 SMILES 预测其 BRICS 子结构序列。这个任务相当于先教模型如何按化学意义拆分分子，而不是直接进入最终逆合成预测。因此模型在微调前已经具备了片段级的反应相关先验。

知识适配器与注入机制
作者没有直接把 KG 表示送进模型，而是先设计了一个任务相关知识适配器。这个适配器通过信息瓶颈思想，保留最有用的知识，同时过滤冗余部分。之后知识被同时注入编码器和解码器，因此能够在理解产物和生成反应物两个阶段共同发挥作用。

主要结果：整体性能
实验结果表明 KnowRetro 在 USPTO-50K 和 USPTO-FULL 上都优于强基线。尤其是在 Unknown reaction class 设定下，Top-1 达到 62.7%；在 Known 设定下，Top-1 达到 71.6%。这说明无标签分子知识确实能够带来稳定收益。

进一步结果：化学有效性与消融
除了 Top-k 结果，作者还报告了 MaxFrag 和 Round-Trip 指标。前者关注主反应物匹配，后者关注生成结果是否能通过正向模型回推到目标产物。结果说明 KnowRetro 不只是更准，也更具化学合理性。消融实验进一步证明，KG 和预训练都是关键模块。

鲁棒性与多样性分析
在知识图中随机删除边或扰动关系后，KnowRetro 的性能下降相对有限，说明方法具有较强鲁棒性。同时，从预测结果的结构相似度分布来看，KnowRetro 生成的候选反应物更加多样，不容易陷入单一模式。

论文总结
总体来看，KnowRetro 的价值不只是把某个指标再提升一点，而是提出了一个更完整的知识引导框架。它证明无标签分子中的层次结构知识是可以被系统利用的，并且这种知识能够在逆合成中转化为更好的预测表现、更强鲁棒性和更高多样性。

审稿意见视角下的优点与不足
从审稿意见看，这篇论文的优点比较明确。审稿人普遍认可它的问题设置有意义，也认可方法有效、实验扎实，尤其是 rebuttal 后补充了多步规划、正向反应、性质预测和更细的消融分析。但是不足也很明确，主要是相对强基线的绝对 Top-1 提升仍偏有限，模块之间的统一设计动机在初稿中表达得不够强，另外公开外部数据集上的说服力还不够充分。因此它是一篇很典型的 borderline paper。

从 KnowRetro 到 DrugKG
如果说 KnowRetro 解决的是“如何把结构知识引入逆合成模型”，那下一步更值得做的是“如何把能量知识也系统引入进来”。因为真实药物设计并不只关心能不能拆，还关心能不能合成、有没有效、是否容易被代谢。也就是说，单纯的结构 KG 只是第一层。

DrugKG 作为统一世界模型
DrugKG 的意义在于，用统一的图结构把结合、代谢和逆合成放在一起，并且给这些关系附上能量语义，比如结合自由能、键解离能和反应能垒。这样图谱就不再只是知识存储，而变成一个支持多任务决策的世界模型。

Energy-KG Core Agent 与闭环优化
在这个设想下，Energy-KG Core Agent 负责统一协调多个子 Agent。Binding Agent 负责疗效，Metabolism Agent 负责代谢负债，Retrosynthesis Agent 负责可合成性。它们都向同一个知识图读写信息，最后形成一个从评估到改写再到验证的闭环优化流程。
